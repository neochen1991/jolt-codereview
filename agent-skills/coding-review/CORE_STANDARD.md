# Coding Core Standard

## 规范说明
只报告通用实现正确性、异常、空值、边界条件、状态一致性和可维护性问题。

## 检查点
- 空值、越界、状态机非法迁移是否被处理。
- 异常是否被吞掉、过宽捕获或直接泄露内部信息。
- 资源是否正确关闭，幂等和重复调用是否安全。

## 如何检查
1. 从新增分支和异常路径开始。
2. 检查输入为空、集合为空、重复请求和失败重试。
3. 只报告会产生错误行为的问题。

## 反例
```java
catch (Exception e) { return null; }
```

## 正例
```java
catch (PaymentException e) { throw e; }
```
